## Overview

A special challenge mode for pre-release publicity event where attendees:
1. Use provided devices with the app pre-installed
2. Login with special exam credentials (`exam1234`)
3. Enter their roll number for identification
4. Complete hardware + software design challenges
5. Get ranked on a public leaderboard

## Authentication Flow

### Step 1: Login with Exam Credentials
```
┌─────────────────────────────────────┐
│           🔐 LOGIN                  │
│  ─────────────────────────────────  │
│                                     │
│  Username: [ exam1234           ]   │
│  Password: [ ********           ]   │

```

- **Special Account:** `exam1234` / `exam1234`
- **Detection:** When this credential is detected → Skip normal app flow
- **Single credential** for ALL devices at event

### Step 2: Roll Number Entry
```
┌─────────────────────────────────────┐
│     📝 ENTER YOUR ROLL NUMBER       │
│  ─────────────────────────────────  │
│                                     │

- **Format Validation:** Must match `1601XXXXXXXX` (12 digits)
- **Differentiation:** Roll number used to identify unique users

| Initial State | **Unlocked** for all users |
| Problem Count | 3 problems per user |

| Sections per Problem | Hardware + Software |

### Problem Statement Generation

- **Dynamic:** Generated by **Gemini 2.5 Flash API**

## Timer System

### Configuration
| Setting | Value |
|---------|-------|
| Total Time | **20 minutes** |
| Display | Countdown visible at top-right |
| Warning | Yellow at 5 min, Red at 1 min |
| Auto-submit | After 20 min, submit current progress |

### Timer UI States
```
Normal:      ⏱️ 18:32    (White/Green)
Warning:     ⏱️ 04:58    (Yellow, pulsing)
Critical:    ⏱️ 00:45    (Red, flashing)
Paused:      ⏱️ PAUSED   (Gray, network issue)
```

### Time Tracking
- **Start Time:** When user enters Challenge 1
- **End Time:** When user clicks final submit OR 20 min expires
- **Duration:** endTime - startTime - pausedTime
- **Paused Time:** Accumulated when network issues detected

### Scoring Example
| Scenario | Timer Shows | Time Used | Notes |
|----------|-------------|-----------|-------|
| Fast completion | 05:00 left | 15:00 | Bonus for speed |
| Normal completion | 00:30 left | 19:30 | Normal scoring |
| Auto-submit | 00:00 | 20:00 | Submit whatever completed |
| Network pause | PAUSED | Timer stops | Fair handling |

## Anti-Cheating System

### Detection Events
| Event | Trigger |
|-------|---------|
| Home Button | `onPause()` + `isFinishing() == false` |
| Recent Apps | `onPause()` + task switch detection |
| Phone Call | `onPause()` + telephony state |
| Screen Off | `onPause()` + screen state |
| App Switch | `onPause()` + package detection |

### Warning System Flow
```
┌─────────────────────────────────┐
│      User presses Home          │
│      or switches app            │
└───────────────┬─────────────────┘
                │
                ▼
┌─────────────────────────────────┐
│   Check: warningCount == 0?     │
└───────────────┬─────────────────┘
                │
      ┌─────────┴─────────┐
      │ YES               │ NO
      ▼                   ▼
┌─────────────┐    ┌─────────────────┐
│ Show Warning│    │ TERMINATE       │
│ Dialog      │    │ SESSION         │
│ warningCount=1   │ status=terminated│
└─────┬───────┘    └─────────────────┘
      │
      │ User continues
      ▼
┌─────────────┐
│ Resume      │
│ Challenge   │
└─────────────┘
```

```

│   └───────────────┘ └───────────────┘  │

### Termination Actions
1. Save progress to Firebase (problem, hardware, software - wherever stopped)
2. Set `status = "terminated"`
3. Set `terminationReason = "cheating" | "phone_call" | "exit_by_choice"`
4. Log timestamp
5. Force navigate to Ranking Dashboard
6. Future logins → Dashboard only (unless admin resumes)

### Ranking Algorithm

```
Final Score = (HardwareScore × 0.4) + (SoftwareScore × 0.4) + (TimeBonus × 0.2)

Where:
- HardwareScore: Gemini evaluation (0-100)
- SoftwareScore: Gemini evaluation (0-100)
- TimeBonus: (20 - timeUsed) / 20 × 100

Example:
- Hardware: 80/100
- Software: 90/100
- Time Used: 15 min

TimeBonus = (20-15)/20 × 100 = 25
FinalScore = (80×0.4) + (90×0.4) + (25×0.2) = 32 + 36 + 5 = 73
```

### Tie-Breaking (Same Score)
1. Lower time wins
2. Higher hardware score wins
3. Earlier submission wins

### Challenge Unlock Notification (Firebase Realtime Listener)

```kotlin
// In RankingDashboardActivity.kt
// Listen for challenge unlock in real-time

val userRef = FirebaseDatabase.getInstance()
    .getReference("preReleaseEvent/participants/$rollNumber/status")

userRef.addValueEventListener(object : ValueEventListener {
    override fun onDataChange(snapshot: DataSnapshot) {
        val canAccessCh2 = snapshot.child("canAccessChallenge2").getValue(Boolean::class.java) ?: false
        val canAccessCh3 = snapshot.child("canAccessChallenge3").getValue(Boolean::class.java) ?: false
        
        if (canAccessCh2 && !hasStartedCh2) {
            showChallengeBanner("Challenge 2 is now available!", 2)
        }
        if (canAccessCh3 && !hasStartedCh3) {
            showChallengeBanner("Challenge 3 is now available!", 3)
        }
    }
})

private fun showChallengeBanner(message: String, challengeNumber: Int) {
    // Show banner at top of ranking dashboard
    challengeBanner.visibility = View.VISIBLE
    challengeBanner.text = message
    startChallengeButton.setOnClickListener {
        navigateToChallenge(challengeNumber)
    }
}
```
